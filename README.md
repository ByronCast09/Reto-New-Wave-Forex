# Relaciones
## Usuario y Pago:
**Relación:** Uno a Muchos (Un usuario puede tener muchas suscripciones)

**Ejemplo:** Si un usuario tiene varias suscripciones a diferentes productos, cada suscripción estará vinculada al mismo usuario.
## Producto y Pago:
**Relación:** Uno a Muchos (Un producto puede tener muchas suscripciones)

**Ejemplo:** Si un producto es una membresía mensual, muchos usuarios pueden suscribirse a ese mismo producto.

## Usuario y AccesoPlataforma:
**Relación:** Uno a Muchos (Un usuario puede tener múltiples accesos a diferentes plataformas)

**Ejemplo:** Un usuario puede tener acceso a Google Classroom, Discord, y Whatsapp, cada uno registrado como un objeto Acceso diferente.
## Usuario y AccesoVideo:
**Relación:** Uno a Muchos (Un Usuario puede tener acceso a muchos videos)

**Ejemplo:** Un usuario pueden tener acceso a muchos videos que le son proporcionados.

## Video y  AccesoVideo
**Relación:** Uno a Muchos (Un video puede dar acceso a muchos usuarios)

**Ejemplo:** Un video da acceso a muchos usuarios para que puedan visulizar.
